# Vendor Invoice Management System

Introduction -

This project aims to develop a comprehensive Vendor Invoice Management tool that facilitates seamless invoice submission, validation, and approval processes. 

Key Objectives - 

1. Instant Information
2. Improved Vendor and Retailer Experience
3. Enhance Auditing Efficiency

Usefulness - 

1. Good Control of Vendor Invoice Processing
2. Efficient Approval Process
3. Efficient Vendor Invoice Tracker
4. Centralized System for Suppliers and the Company

Dataset - 

Created my own dataset. It includes vendor-entered information, purchaser approvals/rejections, and dates of processing. 

KPIs -
1. Vendor Invoice Processing Time
2. Purchaser Approval Time

Files - 
1. VIM.sql: MySQL script containing database and queries.
2. Main.py: file for terminal-supported execution of a streamlit application.
3. Project Design.xlsx - The dataset.
4. Website_Screenshots Folder: Pdf Files of screenshots from local web application

Conclusion -
The Vendor Invoice Management project aims to streamline invoicing processes, optimize vendor relationships, and enhance overall efficiency. By analyzing KPIs and leveraging technology and automation, this project contributes to faster payments, improved vendor relationships, and enhanced financial transparency. It offers a user-friendly and centralized platform for vendors, retailers, and purchasers, fostering collaboration and seamless invoice processing.
